# Registro de cambios de requisitos

**Código:** INI-REQ-CHG-001  
**Proceso:** Adquisición, requisitos, evaluación y gestión de cambios - ISO/IEC/IEEE 12207:2017  
**Responsable:** Marco Ponce de León  
**Estado:** En revisión

Este registro controla cambios que modifiquen el SRS después de su aprobación inicial. Ningún cambio debe implementarse directamente en el código sin registrar su origen, impacto, aprobación y pruebas.

| ID | Solicitud | Requisito afectado | Tipo | Prioridad | Decisión | Versión objetivo |
|---|---|---|---|---|---|---|
| CHG-001 | Modo oscuro | RF-05, RNF-04 | Mejora perfectiva | Alta | Pendiente | v1.1.0 |
| CHG-002 | Completar pestañas Inversión y Aliados | RF-02 | Correctiva/funcional | Alta | Pendiente | v1.1.0 |
| CHG-003 | Adaptar layout a pantallas menores | RNF-03, RNF-04 | Mejora adaptativa | Media | Pendiente | v1.2.0 |
| CHG-004 | Etiquetar datos como demostrativos | Alcance/RNF-05 | Preventiva | Alta | Pendiente | v1.1.0 |
| CHG-005 | Definir alcance del bloque IA Assistant | Alcance | Clarificación | Media | Pendiente | v1.2.0 |

## Condiciones de aprobación

- La solicitud debe tener solicitante y justificación.
- Debe identificar requisitos afectados.
- Debe contar con análisis de impacto.
- El cliente debe aprobar alcance, prioridad y versión.
- QA debe aprobar los criterios de verificación.
- El cierre debe actualizar SRS, arquitectura, pruebas y release.
